# Paper and Tool Policy

## Purpose
Use literature and tools to increase rigor, not to decorate conclusions.

## Literature hierarchy
Prefer:
1. local experiment evidence
2. primary technical sources
3. official model cards / technical reports / docs
4. serious research blogs as methodological commentary
5. secondary summaries only when primary sources are missing

## Tool policy
When planning or validating methods:
- use MCP / skills first if available
- use alphaXiv or equivalent paper lookup when available
- fall back to arXiv, OpenReview, official reports, and official docs

## Mandatory methodological skepticism
Always stay alert to:
- activation patching dependence on corruption and metric choice
- attribution-patching false negatives
- sparse / transcoder / crosscoder faithfulness issues
- long-context visibility confounds
- evaluation stack failure modes
- agent-harness overcomplexity

## Citation discipline for internal notes
Each serious result note should say:
- claim
- evidence type
- source(s)
- why the source matters
- whether it is direct evidence or methodological guidance

## Publication discipline
When planning a paper, optimize for:
- importance
- novelty
- rigor
- clarity
- reproducibility
- strong controls
- a thesis that changes understanding rather than merely adding experiments
